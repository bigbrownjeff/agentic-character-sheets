# agentic-character-sheets

> **Your prompt is a character sheet.** D&D solved persona design in 1974: named slots, exposed mechanics, an evolution log. This is that schema, applied to the agents people actually ship.

See [`voice.md`](./voice.md) for the framing that makes this more than whimsy. The point is legibility: a stat block makes the three things a system prompt hides *visible* — the bounded slots, the exposed dials, and the log of how it got here. "What's the AC of *abusive*?" stops being rhetorical and becomes a number you can argue about.

## Structure

```
schema/        JSON Schema (sheet.json) — the CI gate; a missing slot is a failing check
data/
  sheets/      one JSON array per file; source-of-truth for all characters
  characters.json   merged + sorted build output (regenerated by build)
  adventures.json   the 6 adventures, with party/npc/bestiary id lists
scripts/
  build.mjs    merge + validate sheets/*.json → characters.json; exits non-zero on any error
  serve.mjs    zero-dep static server for site/ on :4500
render/
  render.mjs   SVG card generator (sheet data → card.svg)
  cards/       rendered SVG cards (tracked); PNG exports (gitignored)
site/          static frontend + mirrored data/ (built by build.mjs)
research/      sourcing dossiers for individual characters
SCHEMA.md      field-by-field spec with D&D mapping rationale
STATTING.md    philosophy for assigning scores (what WIS 6 means for an AI agent)
voice.md       the framing — why this is a thesis, not just a meme format
ROSTER.md      full character index grouped by adventure
```

## Data flow

```
data/sheets/*.json
       ↓  npm run build (scripts/build.mjs)
data/characters.json  +  site/data/
       ↓  npm run render (render/render.mjs)
render/cards/*.svg
       ↓  npm run serve
http://localhost:4500
```

`build.mjs` validates every sheet against `schema/sheet.json` rules, checks referential integrity against `adventures.json`, and exits non-zero on any violation. CI runs the same command on every push and pull request — a PR that adds a sheet with a missing slot fails the build.

## Quickstart

```bash
npm run build     # validate + regenerate data/characters.json and site/data/
npm run render    # render SVG cards into render/cards/
npm run serve     # static site at http://localhost:4500
```

No dependencies. Node 20+.

## The Instagram reel — card spec

Vertical **1080×1920**, one persona per card, 6–8 cards per reel (a "class," e.g. *"The Dev Pod,"* *"The Monster Manual,"* *"Ponytail vs his shadows"*). Each card, top to bottom:

1. **Name + Title bar** — IBM Plex Mono eyebrow, Source Serif name. (Matches the site's type system.)
2. **The hex-stat block** — six scores in a 2×3 grid, big numerals, modifier under each. The hero image.
3. **One exposed dial called out** — `AC 19` or `Temp 0.2` in oversized type. The "wait, what's the AC of abusive?" hook.
4. **The dumped stat, highlighted** — Ponytail's `CHA 6 (−2)` ringed in red with the caption *"the dumped stat is the build thesis."* This is the shareable insight.
5. **One Adventure-Log line** — `v1→v2 +2 WIS. Why: shipped less, broke less.` The payoff.

**Caption / hook formula:** *"Your AI agent has a character sheet. It just won't show you the numbers. Here's [Ponytail], the Lazy Senior Dev — 6 Charisma on purpose. 🧵"*

**Series ideas** (ordered for engagement): ① Ponytail (the hook) → ② "every agent you've shipped" (the Working Roster montage) → ③ "the Monster Manual" (Sycophant / Confabulator / DAN — the failure modes as villains) → ④ "build a party, not a PC" (the chorus payoff, links the note).

Rendering: SVG template + the sheet data → static PNG. No design tool needed; the generator is the same idea as the linter, pointed at a stylesheet. Self-host the three site fonts so the cards match the brand (Source Serif / Inter / IBM Plex Mono).
